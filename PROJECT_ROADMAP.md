# Gesture Cube 项目发展方向

## 当前底座

- 摄像头接入：浏览器获取本机摄像头画面。
- 手势识别：基于 MediaPipe Hand Landmarker 识别单手关键点，并输出张开、捏合、握拳、指向等基础手势。
- 视觉核心：Three.js 渲染一个类似科幻能量核心的立方体，跟随手势改变旋转、缩放、颜色和能量强度。
- 响应式界面：适配桌面、宽屏和移动比例，核心 3D 场景全屏铺底，控制面板和摄像头窗口随视口调整。

## 近期可扩展功能

1. 手势指令系统
   - 把手势从“视觉响应”升级为“命令触发”。
   - 示例：捏合确认、张手展开菜单、握拳锁定、指向选择目标。

2. 多模式界面
   - 增加 HUD 菜单、设置面板、数据面板、演示模式。
   - 让魔方不只是视觉物体，而是不同功能模块的入口。

3. 手势校准
   - 为不同用户记录手掌大小、摄像头距离和习惯动作。
   - 降低误判，提升连续控制稳定性。

4. 视觉资产升级
   - 加入更多粒子、后期辉光、能量扫描线、模块展开动画。
   - 可以做成 Jarvis 风格 HUD、宇宙魔方风格能量核心，或企业定制风格。

## 中期方向

1. AI 助手控制台
   - 结合语音识别和大模型，让手势负责选择、确认和导航，语音负责复杂意图。
   - 适合做个人桌面助手、展厅交互系统、智能家居控制入口。

2. 空间交互 Demo
   - 支持双手识别、双手缩放、旋转、拖拽虚拟物体。
   - 可以发展成 WebXR、AR 大屏互动或裸眼 3D 展示底座。

3. 插件化模块
   - 把天气、日程、设备控制、文件检索、数据可视化做成独立插件。
   - 魔方作为统一入口，根据手势切换插件状态。

4. 业务场景模板
   - 展厅讲解：用手势切换产品、楼盘、设备模型。
   - 教育演示：用手势控制 3D 分子、星系、机械结构。
   - 工业监控：用手势查看设备状态、告警、趋势图。
   - 创意装置：用于舞台、发布会、互动屏。

## 长期方向

1. 个人智能操作系统入口
   - 摄像头、语音、键盘、触屏、多设备联动。
   - 目标是一个跨屏的智能控制层。

2. 多端部署
   - Web 端用于快速体验。
   - Electron 用于桌面常驻。
   - 移动端 PWA 用于手机和平板。
   - 大屏端用于展厅和装置。

3. 本地化和隐私增强
   - 手势识别默认在本地浏览器运行。
   - 后续可加入本地模型、本地配置和离线模式，减少隐私风险。

## 推荐下一步

1. 先确定目标场景：个人助手、展厅互动、教育演示、工业看板或创意装置。
2. 为场景定义 5 到 8 个核心手势命令。
3. 增加一个插件面板，让魔方可以打开和切换真实功能。
4. 加入手势校准和调试面板，保证识别稳定。
5. 再做视觉精修和动画包装。
